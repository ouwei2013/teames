# Teames Documentation

This directory keeps project documentation that should not clutter the repository
root.

## Contents

- `releases/` — archived release notes inherited from the Hermes Agent fork.
- `plans/` — implementation plans and design sketches.
- `notes/` — longer product or project notes.

The user-facing documentation site still lives in `website/`. The repository
root keeps only the files that GitHub users and packaging tools expect to find
immediately, such as `README.md`, `LICENSE`, `CONTRIBUTING.md`, `SECURITY.md`,
and runtime entry modules.
